## О проекте

**CROSSVERSE** - стартап/хобби-проект с минимальным бюджетом.

**Домен**: `crossverse.tech` (куплен через Namecheap)

**Миссия**: создать web presence для стартапа с акцентом на экономичность и быстрый запуск.

**Приоритет №1**: запустить контент на домене как можно быстрее для начала SEO индексации.

---

## О владельце проекта

**Технический профиль**:
- Python developer (утилиты, автоматизация, прототипы)
- Базовое понимание веб-разработки без глубокой экспертизы
- Опыт с website builders, поверхностное знание hosting/deployment
- Теоретические знания (Docker, cloud, DevOps) без практического применения
- Комфортно читает технический английский

**Философия**: практичный solo-developer, который не боится учиться новому, но ценит простые решения.

### Принцип экономии токенов ⚡

**Всегда активен** (если не запрошено обратное):
- Краткие ответы без вступлений/заключений
- Только важная информация, минимум очевидностей
- Компактное форматирование
- Примеры только если критичны для понимания

**Отключение**: фраза "дай полный ответ без экономии токенов" или аналогичная.

### Стиль
- Без лишних формальностей
- Прямые ответы на вопросы
- Trade-offs вместо односторонних рекомендаций
- Ирония и сарказм приветствуются :)

## Бюджетная политика ⚠️

**КРИТИЧЕСКИ ВАЖНО**: extremely limited budget (хобби-проект).

### Железные правила

1. **Default = free**: всегда начинать с бесплатных решений
2. **Разовые платежи > подписки**: предпочитать one-time costs
3. **Обязательное исследование** перед любыми тратами:
   - Поиск бесплатных альтернатив
   - Сравнение стоимости/функциональности
   - Обсуждение с владельцем

### Философия

Экономия везде где возможно, но:
- Не в ущерб критической функциональности
- Готовность платить за действительно важное
- Прагматизм > идеализм (иногда $5 экономит 5 часов)

## Workflow для AI агентов

### При работе над задачами

1. **Проверяй бюджет**: есть ли бесплатная альтернатива?
2. **Объясняй просто**: технический английский для терминов, русский для объяснений
3. **Показывай trade-offs**: не просто "используй X", а "X vs Y: плюсы/минусы"
4. **Учитывай уровень**: владелец может изучить новое, но ценит простоту

### Технические решения

- Предпочтение managed services (Vercel, Netlify) vs self-hosting
- Минимальная конфигурация, "works out of the box"
- Документировать важные решения
- Избегать over-engineering
